Contexto y Objetivos

El Escenario de Partida

Banco Singular carece de un sistema de reportes y de acceso directo a su core transaccional. La consolidación de carteras comerciales y ejecutivos depende de procesos manuales basados en archivos CSV estáticos periódicos.

Inexistencia de Reportes Riesgo Alto
No existe una plataforma de visualización directiva. Las carteras se analizan en hojas Excel locales.
Core Aislado y CSVs Estáticos Sin Ingesta Automática
Los datos del core operativo quedan confinados; la integración depende de exportaciones CSV periódicas.
Falta de Gobierno e Integridad Vulnerabilidad
Inexistencia de validación cruzada sobre datos huérfanos y falta de control de accesos centralizado.
Sin Sistema de Reportes Directivo

Arquitectura Target Solvente

Diseñar y desplegar un pipeline de datos automatizado extremo a extremo (ETL), un almacén de datos (MySQL) estructurado y un dashboard gerencial interactivo de alto rendimiento.

Pipeline y Modelo Estructurado Python ETL & MySQL
Procesamiento automático en Pandas, validando integridad referencial antes del commit final.
Visualización Directiva Acelerada Redis local < 15ms
Tableros interactivos Next.js mitigando latencia física del DWH mediante caché Redis.
Acceso Seguro SSO Keycloak OAuth2 / OIDC
Gobierno de identidades centralizado y validación de tokens de seguridad directivos con firmas JWKS.

Enfoque Arquitectónico

Decisiones de Diseño

Lógica Programática

Python ETL vs. Herramientas Visuales (Talend/SSIS)

El uso de Python (Pandas + SQLAlchemy) permite un pipeline 100% programático y versionable en Git. Facilita validaciones en memoria y pruebas automatizadas, eliminando la dependencia y los altos costos de licencias de herramientas ETL comerciales cerradas.

Alternativa Descartada Solución Implementada
ETLs visuales cerrados, dependientes de licencias propietarias y difíciles de versionar. Código modular en Git, automatizado y libre de licencias de software.

Stack Tecnológico Elegido

Python ETL

Normalización y validación programática en memoria con Pandas.

Node.js API

API en Express con validación de seguridad JWKS y control de caché.

Next.js Frontend

Dashboard interactivo con gráficos de barra apiladas en Recharts.

MySQL DWH

Base de datos relacional para el almacenamiento íntegro de carteras.

Redis Cache

Caché en memoria de baja latencia (<15ms) para lecturas optimizadas.

Dokploy VPS

Orquestación y despliegue continuo (CI/CD) automático mediante Docker.

Flujo de Usuario y Arquitectura Lógica Unificada

Estructura en Dokploy (VPS Alemania): Muestra los dos proyectos orquestados en el VPS de Dokploy. El proyecto INTRANET-REPORTES contiene las capas de la aplicación (Frontend, Backend, DB, OIDC y Cache), mientras que el proyecto independiente PROCEDIMIENTOS ejecuta la ingesta programada por Python ETL.
